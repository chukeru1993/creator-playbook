# 知识讲解视频：设计与演出复用框架

适用于技术、商业、方法论等中文知识讲解视频。本框架规定如何把旁白拆成可验证的镜头论证；颜色、字体、尺寸和字幕轨使用项目的视觉规范，例如《DESIGN》。

## 1. 文档职责与优先级

| 文档 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| `video-spec.md` | 当前项目的镜头内容、旁白、时长、素材、精确文案 | 通用字号与组件尺寸 |
| `DESIGN.md` | 默认 token、字体、尺寸、字幕轨、禁用项 | 某个镜头讲什么 |
| 本框架 | 论证结构、空间拓扑、动效语义、素材原则、验收方式 | 具体色号、项目品牌、章节数量 |
| 渲染实现 | 严格实现上述三者 | 自行改写规格 |

冲突时：项目 `video-spec.md` 的镜头事实优先；视觉 token 以 `DESIGN.md` 为准；本框架用于补全镜头组织与验收，不得替代项目规格。

## 2. 核心方法：固定镜头，逐层证明

一个镜头不是 PPT 页，而是一段能在同一全景中完成的论证：

```text
一句命题 -> 一个对象或问题 -> 一组关系/证据 -> 一个判断 -> 可读停留 -> 转场
```

旁白解释因果，画面只保留名词、关系和结论。画面不逐字复述旁白；每出现一个元素，都应能指出它对应哪一个旁白落点。

一个镜头只解决一种关系。新关系无法在现有空间拓扑里清楚表达时，才切镜头。

### 2.1 分镜硬约束：结论优先，切镜例外

`Scene` 的计数单位是**一个观众能带走的最终结论**，不是旁白句子、标题、术语解释、动画动作或章节锚点。

一个宏镜头默认应在同一空间内完成：

```text
命题 -> 对象/证据 -> 关系展开 -> 判断 -> HOLD
```

标题出现后补入对象或证据、同一关系图的展开/聚焦/反例/结论、定义后紧接的同一对象实例、章节标题后直接进入本章第一个概念，以及高信息后的 `HOLD`，默认都是**同一 Scene 内的旁白可见拍**，不得仅因旁白换句而拆镜。

只有同时满足以下两项，才允许新建 Scene：

1. 新内容会留下不同于上一镜的最终结论；
2. 现有空间拓扑无法通过展开、聚焦、变色、替换、缩放或局部重排清楚表达这个新结论。

“旁白换句了”“需要更多变化”“这一拍很短”都不是切镜理由。若当前画面仍能承接下一拍，先把它写入同一 Scene 的旁白可见拍。

## 3. 七种镜头语法与空间拓扑

| 镜头语法 | 叙述任务 | 固定空间拓扑 | 终态 |
| --- | --- | --- | --- |
| 结论 | 先回答“所以呢” | 标题在上/左，结论落在下方 | 一个可复述结论 |
| 分类 | 给观众地图 | 同层类别按阅读方向排列 | 当前类别被标记 |
| 流程 | 解释如何发生 | 起点在左，结果在右；箭头按因果接通 | 完整路径 |
| 层级 | 解释沉淀或依赖 | 基础在下，过渡在中，结论在上 | 顶层被确认 |
| 问题-方案 | 建立价值 | 问题/不完整在左，方案/推荐在右 | Green 方案成立 |
| 对比 | 支持选择 | A 在左，B 在右，中间是关系符号 | 推荐态清晰 |
| 行动 / CTA | 变成下一步 | 行动步骤按顺序排列；提问 CTA 使用主问题 + 两个小问题锚点 | 最小行动或观众问题被保留 |

- 流程、层级、对比的方向是信息含义的一部分，不能为了视觉好看随意翻转。
- 若规格写“从下到上”“左到右”“先风险后方案”，实现必须按该方向验收。

## 4. 单一宏镜头的节奏

镜头的推荐时长由信息量决定：单一判断 `4-7 秒`，普通解释 `8-12 秒`，复杂流程或对比 `12-18 秒`。时长不是硬指标，完整论证才是。

| 阶段 | 比例 | 旁白任务 | 画面任务 |
| --- | --- | --- | --- |
| 预备 | 0-5% | 承接上一句 | 保留锚点，主体压暗或未出现 |
| 命题 | 5-20% | 抛出结论或问题 | 主标题先到 |
| 锚定 | 20-35% | 说出第一个对象 | 主对象、起点或左阵营出现 |
| 建构 | 35-65% | 逐句说明关系 | 每个关系补入一个对象、线或条件 |
| 判断 | 65-80% | 给出风险、推荐或选择 | 中性转为语义状态 |
| 停留 | 80-100% | 让观众消化并收句 | 最终关系保持，再进入转场 |

### 宏镜头预算（预警，不是配额）

- 教程型视频默认按每个宏镜头约 `15-20 秒` 估算；短 Hook、短判断和 CTA 可以更短，复杂关系可以更长。
- 10 分钟教程默认约 `30-40` 个宏镜头。超过 `40` 个时，必须完成“相邻镜头合并审查”；少于 `30` 个时，必须核查是否把多个最终结论塞进一镜。
- 时长不能单独决定切镜。镜头数只用于发现风险，最终以“一个结论是否已经完整证明”为准。

### 对拍规则

1. 旁白先说关系，画面在其后短暂延迟出现；观众先听到，再看到。
2. 一个完整句子最多触发一个主要画面动作。
3. 关键动词才配动作，例如“变成、连接、排除、对比、选择、完成”。
4. 句末重音是状态变化的落点：接通、变绿、变红、锁定或收束。
5. 每引入 `2-4` 个元素，留出一次完整呼吸拍；信息密度来自层次，不来自持续运动。

## 5. 动效语义词典

| 动作词 | 适用对象 | 演出 | 必须留下的终态 |
| --- | --- | --- | --- |
| SLAM | 核心问题、结论 | 快速缩放或推入后立刻落位 | 主命题可读 |
| CASCADE | 同级标签、节点、问题锚点 | 按阅读顺序错开出现 | 全部层级仍可读 |
| LOCK | 当前规则、当前步骤 | 一次描边或状态边锁定 | 当前焦点明确 |
| CONNECT | 流程线、因果关系 | 从起点绘制至结果 | 路径完整 |
| CHECK | 已采用、已完成项 | 单次状态出现 | Green 验收状态 |
| FLAG | 缺失、风险项 | Coral 状态出现或短暂强调 | 风险仍可辨认 |
| DIM | 已被超越的参考、次要项 | 降低亮度或对比度 | 不与当前焦点竞争 |
| RESOLVE | 方案、模板、推荐路径 | 由中性转 Green | 正确路径成立 |
| HOLD | 最终关系 | 停留供阅读 | 不新增信息 |
| FADE OUT | 片尾 | 压暗至黑 | 只用于最终镜 |

- 一个强调动作只改变一个主要维度：颜色、描边、局部缩放或位置，四选一。
- 同一镜头的标题、主对象、辅助关系不能同方向、同速度、同方式进场。
- 术语写进 `video-spec.md` 时，必须同时写明对象和终态，例如“品牌模板 RESOLVE 为 Green 并 HOLD”。

## 6. 真实录屏与素材规则

真实录屏是证据层，程序化元素是解释层。

1. 录屏内的文字、操作和结果保持事实，不改写、不重绘、不伪造。
2. 每个实录镜头先确认：旁白所述对象是否已在原画面真实出现。
3. 覆盖层默认使用边缘标签、短引线、状态词；完整包围框仅在目标坐标稳定且已核对时使用。
4. 一个实录镜头最多一个功能性高亮焦点；不叠多套框、箭头和装饰文字。
5. 静音实录、旁白与字幕必须是独立轨道。录屏无声时，旁白和字幕仍按既定时间轴工作。

## 7. 镜头卡

新项目的每个镜头至少填写以下内容：

```md
### Scene N · <一句核心命题>

- 时长：<由旁白与信息量决定>
- 叙述问题：<观众看完能回答什么？>
- 观众带走的唯一结论：<一句可复述的话>
- 旁白可见拍：
  1. <命题>
  2. <对象/证据>
  3. <关系>
  4. <判断/行动>
- 镜头语法：<结论 / 分类 / 流程 / 层级 / 问题-方案 / 对比 / 行动-CTA>
- 空间拓扑：<如：基础在下 -> token 在中 -> 模板在上>
- 主视觉锚点：<标题 / 数字 / 流程线 / 对比轴>
- 文案锁定：<必须逐字出现的短语；必须转色的词>
- 语义色：<当前焦点、风险、推荐分别是什么，为什么>
- 动效：<对象 + 动作词 + 终态>
- 素材来源：<程序化 / 真实录屏 / 用户图片；时间区间>
- 真实素材覆盖：<允许叠什么；禁止叠什么；目标是否已核对>
- 字幕安全：<主体底边与字幕轨的关系>
- 最终静帧：<观众停留时看到的唯一关系>
- 切镜理由：<新结论 / 新空间拓扑 / 章节反转 / CTA>
- 若不切镜能否继续演出：<否；说明当前拓扑为何不能承接下一关系>
- 禁止项：<不得出现的 UI、人物、框、图标或文案>
- 转场意图：<承接 / 反转 / 章节切换 / 片尾>
```

## 8. 制作与验收流程

1. 先按“观众带走的唯一结论”写宏镜头清单，再为每个结论补旁白可见拍；禁止从旁白逐句切出 Scene。
2. 对相邻宏镜头完成“相邻镜头合并审查”。没有成立的切镜理由，不得进入分镜表。
3. 冻结 `video-spec.md` 的旁白、文案锁定、空间拓扑、素材来源与禁止项。
4. 先搭每个镜头的最终静帧，确认层级、阅读顺序、字幕安全区和真实素材真实性。
5. 再按旁白可见拍补入动作；每一个动作都绑定语义词与终态。
6. 运行静态、运行时、布局、动效和对比度检查。
7. 在根时间线抓取每个子合成的中段与终态帧，逐项对照镜头卡。

### 逐镜头验收清单

- [ ] 画面是否只回答一个问题，并能在静音时复述？
- [ ] 文案锁定的文字是否逐字正确？
- [ ] 空间拓扑是否正确，例如流程左右、层级上下、对比两侧？
- [ ] 状态色是否只表达其规定语义？
- [ ] 最终需要转色、锁定、变暗或保留的对象是否进入正确终态？
- [ ] 实录是否保持真实，覆盖层是否标到核对过的对象？
- [ ] 主体、结论和实录关键区域是否避开字幕轨？
- [ ] 是否没有假网页、假评论区、无目标高亮框、无意义装饰？
- [ ] 是否保留了可读停留，而不是一出现就转场？

## 9. 全片节奏与变化预算

### 相邻镜头合并审查

分镜表冻结前，逐一审查每个 `Scene N -> Scene N+1` 边界，并记录：

| 审查项 | 必须回答的问题 |
| --- | --- |
| 上一镜结论 | `Scene N` 的最终静帧让观众带走什么？ |
| 下一镜结论 | `Scene N+1` 是否留下一个不同、可独立复述的结论？ |
| 拓扑延续 | 保留当前对象、方向和关系，能否通过展开/聚焦继续演出 `Scene N+1`？ |
| 切镜依据 | 若必须切，属于新结论、新空间拓扑、章节反转还是 CTA？ |
| 审查结果 | 能延续则合并，把下一镜改写为当前 Scene 的旁白可见拍；不能延续才保留边界。 |

不接受“多一个画面更有节奏”“旁白进入下一句”“需要一个转场”作为切镜依据。

- 章节可以稳定保留同一锚点、底色、字幕轨与字体，但主体结构必须随论证关系变化。
- 相同主体结构连续使用不超过两个镜头；第三镜必须改用不同语法或不同空间拓扑。
- 高信息镜头后优先在同一 Scene 内通过 `HOLD`、聚焦或收束元素留出呼吸；只有结论改变且现有空间拓扑无法承接时，才另设少元素结论镜。少元素不等于空帧，仍需保留一个明确判断。
- 硬切只用于章节切换、论点反转和风险警告；同一逻辑链优先压暗、焦点切换或短交叉淡化。
- 片尾 CTA 不模拟评论区或点赞 UI；使用观众自身的问题与方法作为最后关系。

## 10. 一句话准则

**让旁白逐步建立关系，让画面严格证明关系；最终静帧只留下一个观众能带走的结论。**
