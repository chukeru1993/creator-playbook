# Graphite Rule Console

面向中文知识讲解视频的 16:9 默认视觉规范。它以 `1920 x 1080` 为基准画布，默认帧率为 `30 fps`；新项目默认直接使用本规范，不为每支视频重新发明字体、字号、间距和状态色。

本文件固定视觉系统，不固定章节数、镜头数、旁白、素材、时长或品牌对象。项目专属内容写入 `video-spec.md`；镜头组织与验收方式见《知识讲解视频-设计与演出复用框架》。

## 1. 使用方式

- 默认目标：中文知识、技术、商业与方法论讲解；横屏 `1920 x 1080`；`30 fps`。
- 非 16:9 项目不直接等比缩放本版布局；应新建该画幅的派生规范，只复用颜色语义、层级关系和动效语言。
- 允许覆盖：项目品牌 logo、品牌主色、经确认的字体、真实素材画面。
- 不允许覆盖：暖近黑基调、状态色语义、字幕轨安全区、章节锚点语法、真实素材不造假的规则。
- 每次开工先在项目 `video-spec.md` 声明：画幅、实际字体文件、是否有真实录屏、是否使用品牌色替换。

## 2. 默认 Token

| Token | 值 | 固定语义 |
| --- | --- | --- |
| Canvas | `#181916` | 全片更深的暖近黑画布；不用纯黑，以抵消导出播放时的提亮感 |
| Panel | `#282824` | 信息面板、低层容器 |
| Ink | `#F4F1EA` | 主标题、核心关系、关键暖白文字 |
| Muted | `#C0BBB1` | 说明、未激活项、次要问题 |
| Hairline | `#4C4942` | 低优先级线、静态边线 |
| Amber | `#F0A93B` | 章节锚点、当前内容焦点、标题竖线 |
| Coral | `#FF6A61` | 风险、错误、不完整输入、反例 |
| Green | `#61D394` | 推荐路径、可执行方案、完成与验收 |
| Blue | `#72A9FF` | 内容属性或第二阵营，不表示状态 |
| Violet | `#A98CFF` | 必要的第三对比阵营，不表示状态 |

### 颜色纪律

- 琥珀色的章节锚点与标题前竖线属于系统 chrome，不计入内容焦点；内容区域仍只能有一个琥珀色焦点。
- Coral、Green、Blue、Violet 都是语义色，不能当装饰色使用。
- 同一对象的状态变化只使用一种主语义色：风险用 Coral，解决用 Green，内容属性才用 Blue 或 Violet。
- 已存在 Coral 或 Green 阵营时，其他容器使用 Panel 与 Hairline，不额外描彩边。

## 3. 固定字体与文字层级

实际渲染必须嵌入或打包字体；没有字体文件时使用下列 fallback，不能让浏览器临时猜字体。

| 用途 | 字体 | 字重 | 字号 | 行高 | 规则 |
| --- | --- | --- | --- | --- | --- |
| 主标题 | `Noto Sans SC`, `PingFang SC`, sans-serif | 900 | 76px | 1.14 | 左对齐；1-2 行；只强调一个词组 |
| 大数字 / 主节点 | `Noto Sans SC`, `PingFang SC`, sans-serif | 850 | 48px | 1.10 | 用于核心对象，不写长句 |
| 节点 / 次标题 | `Noto Sans SC`, `PingFang SC`, sans-serif | 850 | 38px | 1.12 | 流程节点、层级名称 |
| 问题锚点 / 说明 | `Noto Sans SC`, `PingFang SC`, sans-serif | 700 | 34px | 1.22 | 每项尽量一行 |
| 章节锚点 | `IBM Plex Mono`, `JetBrains Mono`, monospace | 700 | 22px | 1.00 | 固定在左上 |
| 标签 | `IBM Plex Mono`, `JetBrains Mono`, monospace | 700 | 18px | 1.00 | 对象分类、短状态 |
| 元数据 | `IBM Plex Mono`, `JetBrains Mono`, monospace | 700 | 16px | 1.00 | 编号、短注释 |
| 字幕 | `Noto Sans SC`, `PingFang SC`, sans-serif | 650 | 30px | 1.30 | 居中；关键词使用语义色 |

- 不使用负字距。
- 标题、节点与说明的字号不因一句文案过长而变小；先缩写文案，再考虑换行。
- 英文或数字与中文混排时仍使用正文体系；编号与短标签才使用等宽字体。

## 4. 画布、留白与字幕轨

| 区域 | 固定值 | 用途 |
| --- | --- | --- |
| 内容左右安全边距 | 96px | 标题、图表与面板的左/右边界 |
| 顶部安全边距 | 50px | 章节锚点起点 |
| 内容底部预留 | 154px | 主体和结论不得进入此区域 |
| 章节锚点竖线 | `4 x 28px` | 左上系统识别 |
| 标题竖线 | `8px` 宽 | 核心命题前的琥珀标记 |
| 面板状态边 | `3px` 左边线 | 仅表达状态或路径 |
| 面板圆角 | 4px | 信息容器；不得做大圆角卡片 |
| 字幕左右边距 | 120px | 字幕底板宽度 |
| 字幕底距 | 42px | 字幕轨位置 |
| 字幕最小高度 | 76px | 单行与双行的稳定容器 |
| 字幕圆角 | 6px | 仅限字幕底板 |

- 内容容器应使用 `padding`、flex 或 grid 推导，不用一组无语义的绝对像素坐标拼版。
- 结论胶囊、流程终点和行动按钮必须位于内容底部预留线之上；字幕始终是最底层的阅读轨。
- 需要强调的面板可以用 3px 状态左边线；不默认加完整彩色描边或阴影。

## 5. 固定画面语法

### 5.1 三层画面

每个程序化镜头至少包含三层：

1. 背景：仅使用 Canvas 纯色画布，不叠加幽灵词、局部暗光或纹理。
2. 主体：一个完整的论证结构，而不是网页式居中卡片。
3. 前景：章节锚点、短线、标签、状态边或编号。

背景保持静止；Canvas `#181916` 是全片唯一的背景色。章节锚点、主体关系与前景状态承担全部视觉层级。

### 5.2 章节与命题

- 左上固定：琥珀短竖线 + `[章节序号] / [章节总数]` + 极短章节名。
- 每个核心命题前固定一根琥珀标题竖线；标题主体保持 Ink。
- 主体优先占据中部大面积宽度。标题、图形和关系线共同构成一幅论证图，不把整个镜头做成一张网页卡片。
- 一个镜头只表达一种关系：结论、分类、流程、层级、问题-方案、对比、行动或提问式 CTA。

### 5.3 容器与焦点

- 面板是承载真实对象的容器，不是装饰；深色实底、低对比边线、4px 圆角、无强阴影。
- 禁止层层套卡、悬浮仪表盘、玻璃拟态、无意义的大面积分割线。
- 同一画面最多一个主要状态转变：变色、描边、局部缩放或位置变化四选一。
- 不连续两镜使用同构的卡片网格；章节锚点可以稳定，主体结构必须随关系变化。

## 6. 真实录屏与素材覆盖

真实录屏、截图或用户提供视频是事实层，不是可重绘的背景。

- 原始素材必须保持可辨识的真实内容；不替换文字、不伪造 UI、不添加虚假验收结果。
- 覆盖层只解释旁白，每个实录镜头最多一个功能性内容焦点。
- 包围框、黄框或指向线只能标注已经核对过的真实目标区域；坐标不可靠时，使用边缘标签、短引线或状态词，不画大矩形。
- 覆盖层不得遮挡操作主区域、鼠标动作、字幕轨或录屏中的关键结果。
- 实录音频、项目旁白和字幕各自是独立轨道；静音实录不得替换旁白。

## 7. 固定动效语言

每个宏镜头默认遵循：`命题 -> 锚定 -> 建构 -> 判断 -> 停留`。

| 对象 | 默认进入 | 强调 | 终态 |
| --- | --- | --- | --- |
| 章节锚点 | 0.30s 轻推进入 | 无 | 整镜停留 |
| 主标题 | 0.46s 快速落位 | 关键词一次局部强调 | 保持 |
| 节点 / 面板 | 0.30-0.40s 按叙述方向进入 | 状态左边线或文字变色 | 保持 |
| 流程线 | 0.56s 从起点绘制 | 接通终点 | 保持 |
| 状态词 | 0.24-0.32s 出现 | Green/Coral 变色 | 保持 |
| 结论胶囊 | 0.30s 轻微回弹 | 一次 Green 锁定 | 保持 |
| 最终镜 | 0.60s 压暗至黑 | 无 | fade out |

- 同一镜头的标题、主对象与辅助关系必须有不同的进入方向或重量。
- 动效只在旁白的关系词或句末重音处发生；普通解释不逐句飞入。
- 背景不使用环境动效；动效只服务于标题、主体关系和前景状态。
- 相邻镜头使用压暗、焦点切换或短交叉淡化；只有章节切换、反转和风险警告使用硬切。

## 8. 禁止项

- 纯黑/纯白背景、满屏网格、全屏单色蓝、黑色扫描线转场。
- 幽灵文字、局部光晕、霓虹、粒子、黑金炫技、全屏线性渐变、毛玻璃、3D、人物和无关 B-roll。
- 假网页、假录屏、伪造评论框、点赞图标、无来源图表。
- 无目标的大矩形高亮框、所有卡片彩边、每句话同方向飞入、逐字打字机。
- 固定项目文案、项目品牌名、章节数、镜头时长、素材路径或字幕条目。

## 9. 项目验收

交付前逐项确认：

- [ ] 实际字体、色值、字号、面板圆角和字幕轨符合本文件。
- [ ] 章节锚点和标题竖线存在，但内容区只有一个琥珀焦点。
- [ ] Coral、Green、Blue、Violet 都有明确语义，没有被当成装饰。
- [ ] 主体没有进入字幕安全区；结论胶囊与字幕不重叠。
- [ ] 真实素材没有被伪造，覆盖层标注了可验证的真实目标。
- [ ] 背景、主体、前景三层均存在，且背景不抢主体。
- [ ] 静态关键帧与播放过程都可读；没有把最终关系藏在转场中。
