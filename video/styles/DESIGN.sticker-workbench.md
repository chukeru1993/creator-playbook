# 奶油纸面贴纸工作台（Sticker Workbench）

> 适用内容：面向中文 AI、自动化、产品方法论与工具解释的横屏知识视频。
>
> 画幅与帧率：`1920 × 1080`、16:9、`30 fps`。

本文件固定一种“贴在奶油色画布上的可动解释板”风格：**大面积留白 + 粗黑描边 + 有重量的硬投影 + 高饱和纸片组件**。它适合把抽象的 Agent、提示词、流程、决策和数据关系，转为一眼看懂的实体对象。

它不是 UI 还原规范。项目专属旁白、工具界面、人物、品牌、时长、章节和镜头内容写进项目自己的 `video-spec.md`；真实截图仍按事实素材处理。

## 1. 使用边界

- 默认适用：中文技术讲解、AI Agent 机制、自动化流程、产品概念与方法论；横屏 `1920 × 1080`。
- 核心感受：像一张被整理过的纸面工作台，而不是 SaaS 仪表盘、PPT 或赛博 HUD。
- 每镜命题：用 1–3 个显性物件讲完一个关系，例如“问题卡 → 处理窗口”、“三个角色 → 一个流程”、“输入 → 判断 → 输出”。
- 允许覆盖：已确认的项目 logo、经授权的真实素材、品牌名、实际字体文件；真实录屏可以放入“窗口”容器，但不得被重绘成假界面。
- 不允许覆盖：暖米白大底、近黑粗描边、硬投影、充足留白、颜色按对象分配的规则、字幕安全区和真实性规则。
- 项目开工确认：实际可嵌入字体、是否出现真实录屏、是否有品牌色替换、是否需要字幕，以及“彩色卡片代表对象还是状态”。
- 非 16:9 项目不得直接缩小本布局；可保留色彩、描边和组件语法，但应为该画幅新建派生规范。

## 2. 设计 Token

| Token | 值 | 固定语义 |
| --- | --- | --- |
| Canvas | `#F5F0E6` | 全片静止的暖米白纸面；不得改成纯白或冷灰 |
| Paper | `#FFFDF7` | 窗口、文档、标签的内页 |
| Ink | `#1D1D20` | 文字、描边、关系线、图标；近黑而非纯黑 |
| Ink Soft | `#4D4A48` | 次级说明、非焦点图形 |
| Line Soft | `#C9C0B2` | 非关键引线、低优先级分隔与虚线 |
| Blue | `#3E7BF2` | 主角色、主工具、信息输入或第一阵营 |
| Pink | `#F13B88` | 第二角色、输出、对照对象或第二阵营 |
| Orange | `#FF9F1C` | 问题、待办、关键问题卡或需要注意的对象 |
| Lime | `#B9E833` | 当前动作、通过、确认、短提示和连接焦点 |
| Violet | `#B99AF7` | 仅在三方关系中使用的第三对象；不能当状态色 |
| Shadow | `#2A292B` | 所有硬投影；保持单一色值，不能改为模糊阴影 |

### 颜色纪律

- Canvas、Paper、Ink 是每镜的固定底层；彩色只落在有叙事职责的组件上，不给普通容器“上色”。
- 蓝、粉、橙、紫优先表示**不同对象或阵营**，而不是“好/坏”的装饰；一个对象从头到尾保持同色。
- Lime 只用于动作已触发、重要提示、连接点、确认或单个小标签。它是稀缺的阅读焦点，不能铺满卡片。
- 一镜通常使用 1 个主色对象 + 1–2 个辅助对象；超过 3 个高饱和大色块时，必须证明这是理解关系所必需的。
- 没有“彩色描边”系统：彩色组件一律用 Ink 描边和 Shadow 硬投影；普通 Paper 面板用 Ink 描边即可。
- 不使用渐变、透明玻璃、发光、霓虹、金属高光或把颜色仅当氛围的暗色背景。

## 3. 字体与文字层级

实际渲染必须嵌入字体；若无字体文件，使用以下明确 fallback。中文以清晰厚重的无衬线为主，英文短词可更圆润，但不能抢过中文命题。

| 用途 | 字体 | 字重 | 字号 | 行高 | 规则 |
| --- | --- | --- | --- | --- | --- |
| 主命题 | `Noto Sans SC`, `PingFang SC`, sans-serif | 900 | 72px | 1.12 | 1–2 行；Ink；左对齐或置于窗口中央；只强调一个词组 |
| 大数字 / 大问号 | `Fredoka`, `Noto Sans SC`, sans-serif | 800 | 92px | 0.95 | 只放一个核心数字、符号或极短英文词 |
| 卡片标题 / 关键词 / 结果 | `Noto Sans SC`, `PingFang SC`, sans-serif | 800 | 40px | 1.15 | 适合对象名、流程节点、结论和必须一眼读懂的短词 |
| 正文 / 说明 | `Noto Sans SC`, `PingFang SC`, sans-serif | 700 | 30px | 1.35 | 每张卡最多 2–3 条；不用密集段落 |
| 中文短标签 | `Noto Sans SC`, `PingFang SC`, sans-serif | 800 | 28px | 1.10 | 用于卡片顶部的分类、问题和状态；它仍是知识内容，不得伪装成 UI 微文案 |
| 英文短标签 / 窗口 chrome | `Fredoka`, `Arial Rounded MT Bold`, sans-serif | 700 | 24px | 1.00 | 大写可加 `0.06em` 字距；只用于短标签 |
| 编号 / 元数据 | `IBM Plex Mono`, `JetBrains Mono`, monospace | 700 | 20px | 1.00 | 只放章节号、时间码、版本或不影响理解的极短流程号 |
| 字幕 | `Noto Sans SC`, `PingFang SC`, sans-serif | 750 | 32px | 1.28 | 居中；Ink 字配 Paper 底板；关键词可用当前对象色 |

- 中文主标题不用花体、斜体、描边字或“逐字弹出”；它负责迅速读懂。
- 英文短词可使用圆润粗体制造玩具感，但不用任何字体模拟手写涂鸦。
- 中文语义文字（对象名、问题、规则、输入、输出、风险、结果）不得使用“英文短标签 / 窗口 chrome”或“编号 / 元数据”层级；卡片内最小使用“正文 / 说明”层级。
- 不使用负字距。文案过长时先改写为短命题、扩大卡片或拆成两层卡片，不以缩小字号、压缩字距补救。
- 播放验收以 `1920 × 1080` 成片的常规预览大小为准：观众应在 `2s` 内读出所有语义文字；读不出时先删减或放大，不能降级为 UI 微文案。
- 强调优先采用色块、贴纸标签或组件位置变化；同一句不同时使用加粗、彩色、下划线、描边和放大。

## 4. 画布、留白与字幕安全区

| 区域 | 固定值 | 用途 |
| --- | --- | --- |
| 外框 | `10px` Ink | 全片四周的可见印刷边框；内容不得与其相撞 |
| 内容左右安全边距 | `120px` | 主命题、物件、流程线的最小边距 |
| 顶部安全边距 | `74px` | 贴纸角标、窗口标题栏与标题起点 |
| 内容底部预留 | `168px` | 主关系、结论和真实素材不得进入 |
| 主体留白目标 | 不少于画面 `35%` | 让 1–3 个贴纸物件有呼吸感，不排成满屏 UI |
| 组件间最小间距 | `44px` | 关系线与硬投影不会相撞 |
| 窗口 / 文档圆角 | `10px` | 仅用于大 Paper 容器；不做胶囊化界面 |
| 贴纸圆角 | `8px` | 小色块和便签；大问号卡可到 `18px` |
| 标准描边 | `4px` Ink | 面板、贴纸、角色卡、流程节点与图标 |
| 标准硬投影 | `10px 10px 0 #2A292B` | 只向右下投射；不可模糊、不可半透明 |
| 字幕左右边距 | `260px` | 留出左、右叙事物件和硬投影 |
| 字幕底距 | `42px` | 字幕稳定在外框之上 |
| 字幕最小高度 | `76px` | Paper 底板；最多两行 |

- 大多数镜头采用不对称的左右双主体：左侧是“问题 / 输入 / 人”，右侧是“系统 / 结果 / 方案”；它们之间用一根明确关系线连接。
- 单一主物件可居中，但应配合角标、小贴纸或轨迹，避免落成空白 PPT。
- 全片可固定外框，禁止每镜出现不同的背景、满屏边栏或网页导航。
- 用 flex、grid、锚点和具名组件推导布局；避免把每张贴纸堆成无语义的绝对坐标。
- 字幕轨是最后的阅读层。任何底部卡片、流程终点、角色脚部或注释都必须让开它。

## 5. 固定画面语法

### 5.1 三层画面

1. **背景**：Canvas 纯色 + Ink 外框。默认静止、无网格、无渐变、无粒子、无环境光。
2. **主体**：1 个完整的“纸面论证”。主体可以是窗口、文档、问号卡、角色卡、流程节点、徽章或环形关系，但必须读得出对象之间的因果或归属。
3. **前景**：极少量短标签、Lime 提示、路径箭头、小星形或编号。前景用于引导视线，不能盖住主体文字。

### 5.2 基础组件

| 组件 | 构造与用途 | 禁止用法 |
| --- | --- | --- |
| 浏览器 / 文档窗口 | Paper 底、4px Ink 描边、10px 圆角、10px 硬投影；可有 Blue/Pink 标题条和 2–3 个小圆点。装真实工具画面或抽象规则。 | 伪造真实产品的 UI、堆满输入框、把整镜做成后台界面。 |
| 便签 / 贴纸标签 | Blue、Pink、Orange 或 Lime 实底；4px Ink 描边；8px 圆角；可轻微旋转 `-5°` 至 `5°`。用作短问题、判定、注释。 | 用来承载段落、每个元素都旋转、叠成不可读的装饰墙。 |
| 问题卡 | Orange 或 Paper 的竖向卡片，中心只放大问号、眼睛、感叹号或 1 个短问题。用于“未知 / 待判断”。 | 同时放大问号、长标题、列表和多个图标。 |
| 角色卡 | 有色圆角矩形或小方块，可有极简表情、眼镜、徽章和短名称。用于抽象参与者，不要求拟真。 | 人物照片、3D 角色、复杂肢体、无关的可爱装饰。 |
| 流程节点 | 纸片、窗口或角色卡只可通过 5.2.1 规定的关系线相连；线条只说明方向或关系。 | 无箭头含义的蜘蛛网、细灰线、复杂 BPMN 图。 |
| 结果章 / 徽章 | Lime 小标签、圆形印章或低矮贴纸。只在判断完成时出现。 | 每张卡都加“成功”章，或用它替代结论。 |

### 5.2.1 关系线约束（强制）

关系线是用来读懂因果、流程、归属或坐标的**关系层**，不是装饰。全片只能使用以下两套语法；同一条关系不得临时改色、加粗、换箭头或改成第三种样式。

| 关系类型 | 视觉语法 | 使用边界 |
| --- | --- | --- |
| 流程 / 因果 | `4px` Ink 实线 + 单个小型 Ink 实心箭头 | 只用于有明确先后或流向的关系，例如输入→判断→输出、问题→解决、角色→处理器。 |
| 坐标 / 对照 / 定位 | `4px` Ink 主轴 + `3px` Line Soft 虚线辅助轴；默认无箭头 | 只用于对照、坐标、辅助定位或解释真实素材的短引线。若没有方向含义，禁止添加箭头。 |

- **颜色纪律**：Lime 只用于“已接通 / 已确认”的连接焦点或节点，Orange 只用于风险路径所连接的对象；两者不能制造新的线型、彩色箭头或整段彩线。
- **图层纪律**：关系线永久位于卡片、节点、文字、标签和硬投影之下；它可以经过画布留白，但不得穿入卡片、遮住文字、被硬投影截断，或看起来像断线。
- **端点纪律**：线的起点和终点停在节点 Ink 描边外的留白中，与描边和硬投影至少留出 `8px` 安全间隙。先通过锚点、路径转折或调整组件位置解决冲突，不能把端点插进卡片、文字区或投影中。
- **箭头纪律**：在 `1920px` 画布上，箭头使用固定小尺寸，不随线宽或节点大小放大。SVG 推荐 `markerUnits="userSpaceOnUse"`、`16 × 16` marker、约 `14px` 的实心三角；每条流程线最多一个终点箭头。
- **时序纪律**：源节点与目标节点必须先清晰可见；目标节点完成落位后，才沿阅读方向绘制该段关系线，最后才弹出 Lime 连接焦点。不能因关系线容器提前进入，让后续箭头先于内容出现。
- **SVG 实现纪律**：初始 DOM 禁止写 `marker-end`。路径开始绘制的同一时间点才设置 `marker-end`（例如 GSAP 的同一时间点 `.set()`）；这样即使路径用 `strokeDashoffset` 隐藏，也不会留下悬空箭头。

### 5.3 构图模式

- **问题 → 解决**：Orange 问题卡在左，Paper/Blue 窗口在右；中间由 Lime 点或箭头接通。
- **多角色协作**：最多 3 张角色卡半覆盖地围绕一个窗口或中心节点；每张仅保留名称、色彩与一项职责。
- **输入 → 判断 → 输出**：三个大小不完全相同的纸片沿一条明确路径排布；中间“判断”最大，输出用彩色贴纸锁定。
- **对照**：左、右两张不同色的同类卡片隔开足够空白，中间放一个小问号、眼睛或短连接词；不用传统表格。
- **概念展开**：中心大窗口 + 2 个轻微倾斜的卫星贴纸；卫星物件位于中心物件外缘，不与正文重叠。

### 5.4 焦点纪律

- 每镜只有一个需要立即读懂的“主物件”；另外 1–2 个物件只帮助解释关系。
- 所有组件都可有描边和硬投影，但一镜只有一个发生关键色变、缩放或旋转的状态改变。
- 角标、标题条、小星形、Lime 点属于引导线索，不能与主物件争夺面积或饱和度。
- 不连续两镜沿用同一套“双卡并排”构图；当关系从问题变成流程或对照时，主体结构也应改变。

## 6. 真实素材规则

真实录屏、截图和用户素材是事实层；本风格只给它们一个“纸面窗口”，不重画或美化成虚构界面。

- 真实素材放入 Paper 窗口时，保留可辨识的原始文字、数据、时间、状态和操作结果。
- 可在窗口外添加 1 个彩色贴纸标签、1 条短引线或 1 个 Lime 提示点，解释旁白正在说的真实区域。
- 录屏内的原始 UI 配色不必被强行改成 Blue/Pink/Orange；贴纸风格只存在于外层框架。
- 坐标未核对时不画大红框。优先使用边缘标签、短引线或“此处需要确认”的 Orange 问题卡。
- 禁止伪造窗口内容、改写真实数据、添加不存在的成功提示、遮挡关键操作或把原视频水印/频道标识迁移到新项目。

## 7. 动效语言

动效应像把纸片依次贴到桌面上：**有弹性、有方向、停下后稳定**。静态背景不参与环境动效；动效只发生在阅读顺序与关系建立的位置。

| 对象 | 默认进入 | 强调 | 终态 |
| --- | --- | --- | --- |
| Ink 外框 / 固定小角标 | `0.18s` 淡入 | 无 | 全镜稳定 |
| 主窗口 / 主卡 | `0.34s` 从叙事方向平移 `40–64px`，同时由 `0.86` 弹至 `1` | 一次轻微 `1.00 → 1.035 → 1.00` | 保持水平、稳定可读 |
| 倾斜贴纸 | `0.28s` 从 `0.76` 缩放并回弹，最终落在 `-5°..5°` | 仅一次 `2°` 摇摆或局部放大 | 停止，不持续抖动 |
| 角色卡 | `0.30s` 从下方或侧方跃入 | 眼镜、标签或表情可延迟 `0.10s` 出现 | 保持，不能持续跳动 |
| 流程线 / 虚线轨迹 | 在关联节点落位后，用 `0.42–0.56s` 沿阅读方向绘制 | Lime 连接点在接通时弹出 | 保持完整关系 |
| Lime 提示 / 结果章 | `0.20–0.26s` 快速弹入 | 一次 `1.12 → 1.00` 锁定 | 保持到结论读完 |
| 章节转场 | `0.18–0.28s` 快速淡化或硬切后重组 | 仅在观点反转、风险警告时硬切 | 不做整页翻转 |
| 最终结论 | `0.42s` 主结论卡落位，辅助贴纸后入 | Lime 徽章最后锁定 | 停留至少 `1.2s` 后淡出 |

- 动效只配合“所以、但是、交给、判断、完成”等关系词或句末重音，普通解释不逐句飞入。
- 主要对象的进入方向要体现因果：输入从左/上进入，处理器居中落位，输出向右/下出现；不要每个元素都从下方弹入。
- 使用有界的弹性曲线，例如 `cubic-bezier(.34, 1.56, .64, 1)`；不可使用长时间果冻、无休止悬浮、随机抖动或转圈。
- 相邻镜头应保留一个视觉锚点（外框、某个对象色或同一窗口语法），再替换主体关系；避免用炫目的转场代替叙事。

## 8. 禁止项

- 纯白、纯黑、冷灰或全屏渐变背景；扫描线、网格、粒子、镜面反射、霓虹发光、玻璃拟态、毛玻璃和 3D。
- 细描边、柔焦阴影、低对比灰字、无边框扁平卡片、规则的等宽卡片网格、满屏 dashboard。
- 照片人物、写实机器人、复杂插画、无关 B-roll、仅为填满留白的图标雨。
- 假网页、假录屏、假评论、伪造指标、原参考视频的频道 logo、水印、角色命名、原片文案或界面截图。
- 每句话一个弹窗、逐字打字机、全元素同方向飞入、持续摇摆、全屏翻页、故意制造眩晕的缩放。
- 把所有彩色组件做成相同大小、相同角度、相同权重；本风格的活力来自有主次的不规则纸片，而不是杂乱。

## 9. 项目验收

- [ ] 画布为 Canvas 暖米白，四周 Ink 外框存在；背景没有渐变、纹理、发光或无意义装饰。
- [ ] 所有纸片/窗口使用近黑 `4px` 描边和同向、不模糊的硬投影；没有柔和投影或彩色描边。
- [ ] 每镜只有一个主物件，彩色大色块不超过叙事所需数量，Lime 只表示当前动作或确认。
- [ ] 至少保留约 `35%` 的有意留白；主体、硬投影与结论均避开字幕安全区。
- [ ] 文字层级遵守字号下限，卡片文字可在一遍播放中读完；没有用缩小字体补救过长文案。
- [ ] 卡片内所有中文语义文字至少为 `30px / 700`；对象名、规则、关键词和结果至少为 `40px / 800`。`20px` 等宽字只出现于不影响理解的元数据，未被用于知识内容。
- [ ] 窗口、便签、角色卡和关系线确实说明一个完整关系，而不是网页组件拼贴。
- [ ] 每条关系线符合 5.2.1 的两套语法之一；线层在节点、文字和硬投影下方，端点与组件描边/投影至少保持 `8px` 间隙，没有遮挡、断连或突兀的大箭头。
- [ ] 对每条带箭头的流程线，检查“目标节点出现前、路径绘制中、路径完成后”三帧：箭头只能随路径开始出现，任一帧都没有无来源、无目标或悬空的箭头。
- [ ] 真实素材保持事实可辨，覆盖层只标注已核对对象；未出现伪造 UI 或未经授权的原片元素。
- [ ] 进入、连接、锁定和停留的顺序与旁白关系一致；播放过程与关键静帧都可读。
