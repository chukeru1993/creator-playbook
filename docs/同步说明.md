# 同步说明

## 首次关联 GitHub

在仓库根目录执行：

    git add .
    git commit -m "chore: initialize creator playbook"
    git branch -M main
    git remote add origin <你的 GitHub 仓库地址>
    git push -u origin main

本地仓库已经初始化；只需补充远程地址并完成首次提交。

## 每次推送前检查

- [ ] 新内容是可复用 Skill、样式规范或分镜方法，而不是某个项目的交付物。
- [ ] 新增或更新的文件已经登记到 资产索引。
- [ ] 根目录的 MIT LICENSE 文件存在，且新增内容可以按 MIT 许可发布。
- [ ] Skill 目录包含它运行所需的全部文件和元数据。
- [ ] Markdown 链接、相对路径和模板占位符均已检查。
- [ ] 没有提交录屏、音频、导出文件、密钥、私密信息或无授权素材。

## 推荐提交粒度

一次提交只解决一个明确的资产变化，例如：

    skill: refine hook audit
    styles: add vertical video variant
    storyboard: improve scene-card checklist

不要将 Skill 迭代、样式大改和无关格式化混在一次提交中。
